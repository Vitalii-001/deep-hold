# Deep Hold

HTML5 idle/incremental гра «підземна імперія гномів» для веб-порталів (CrazyGames, Poki). Комерційна мета — revenue share з реклами на порталах.

## Статус

- **Дизайн затверджено:** `docs/superpowers/specs/2026-07-10-deep-hold-design.md` — єдине джерело правди по скоупу і механіках.
- **v1 імплементовано** за планом `docs/superpowers/plans/2026-07-10-deep-hold-v1.md`.
- **Cartoon UI Phase 1 / Sub-phase 1 завершено:** shell, light-cartoon дизайн-система, `GameShell`, panel navigation, ель-метр, адаптація онбордингу й `src/assets/ART_CONTRACT.md`.
- **Cartoon UI Phase 1 / Sub-phase 2 завершено:** `OverviewPanel` з Goal-card, `MilestonesPanel`, `StatsPanel`, дефолтна панель `overview`.
- **Cartoon UI Phase 1 / Sub-phase 3 завершено:** `SettingsModal`, `ConfirmationModal`, персистентні UI-налаштування (`deep-hold-ui-settings`), тумблери particles/high-contrast/compact, перенесення Reset у Settings, volume-слайдери.
- **Наступний крок:** owner visual/playtest pass, тюнінг балансу (`src/config/`), потім підготовка до публічного демо/itch.io.

## Геймплейний орієнтир

- По ядру idle/incremental-геймплею орієнтуємося на **Mister Mine**: вертикальний прогрес у шахті, довга економічна крива, шарова структура, робітники/будівлі/апгрейди, регулярні goals і зрозумілий “next thing to do”.
- Deep Hold не є копією Mister Mine. У нас власний дизайн, власна тема гномів, власні назви, баланс, progression-модифікації, UX-рішення й portal-first обмеження.
- Використовувати Mister Mine як жанровий benchmark для відчуття ядра: pacing, ясність наступної цілі, unlock cadence, довжина сесій, потреба в overview/stats/goals. Не переносити чужі ассети, UI-композиції, тексти чи точні формули.

- **Обраний арт-напрям:** #1 «light cartoon» — мокапи в `docs/design/proof_design_1.png` (2/3 — відхилені альтернативи). Слідувати цьому вигляду.
- **Дизайн-токени й layout:** `docs/superpowers/specs/2026-07-13-cartoon-ui-phase1-shell-design.md`. Кольори насемплено з мокапа; UI-панелі — світлий parchment + золото/дерево-акценти.
- **Принцип:** структуру, розкладку й кольори повторювати за мокапом якомога точніше через CSS; фінальну «як на мокапі» фактуру дають растрові **WebP-асети пізніше** — шляхи/іменування в `src/assets/ART_CONTRACT.md`. До асетів — плейсхолдери й наявні іконки.
- **Візуальна верифікація:** я не бачу власний CSS напряму — фідбек по вигляду дає користувач (скріншоти) або скріншот-петля через headless-браузер, якщо налаштована.

## Ключові рішення (деталі в спеці)

- Стек: Vite + React + TypeScript + Zustand; тік-система на requestAnimationFrame; збереження в localStorage з офлайн-прогресом.
- Мова гри — англійська. Спілкування з користувачем — українською.
- Внутрішній бюджет білда < 5 МБ (значно суворіше за реальні ліміти CrazyGames — див. розділ «Вимоги порталу CrazyGames»).
- Баланс — в окремому конфігу, не хардкодити в компонентах (тюнінг балансу — головна ітераційна робота жанру).
- Іконки: game-icons.net (CC BY — вести список авторів для credits).
- Престиж, IAP, локалізації — свідомо поза скоупом v1.
- Перевірки: не ганяти `npm run test` / `npm run build` після кожної CSS/layout-ітерації. Запускати їх, коли змінюється core logic, save/load, конфіги балансу, типи/контракти, або перед фінальним checkpoint задачі.

## Контекст користувача

- Фронтенд-розробник, впевнений у React/Next.js і Stripe; темп — 10–20 год/тиждень.
- Роль користувача в розробці: ревью, плейтест і рішення по балансу/відчуттях геймплею.

## Вимоги порталу CrazyGames (обов'язкові для релізу)

Джерела: docs.crazygames.com — [gameplay](https://docs.crazygames.com/requirements/gameplay/), [technical](https://docs.crazygames.com/requirements/technical/), [ads](https://docs.crazygames.com/requirements/ads/). Звірено 2026-07-17.

### Технічні
- **Розміри бандла:** загалом ≤ 250 МБ, ≤ 1500 файлів; **initial download ≤ 50 МБ** (хард-кап публікації), **≤ 20 МБ** — щоб потрапити на мобільний homepage; час до геймплею ≤ 20 с. (Наш внутрішній бюджет `< 5 МБ` суворіший — тримати його.)
- Initial download міряється від старту завантаження до події SDK `Gameplay start`.
- **Тільки відносні шляхи** до файлів бандла (абсолютні не завантажаться).
- **Адаптивний iframe 16:9**, landscape на десктопі; портрет — лише з чорними полями/фоном. Контент читабельний при **devicePixelRatio = 1**; діапазон від 800×450 (мобілка) до 1920×1080 (десктоп fullscreen) → принцип «вміщатися у viewport без скролу сторінки».
- **Не робити власну кнопку fullscreen** — портал додає її сам.
- Фізика/симуляція стабільна на різних refresh rate (144/165 Гц): тік прив'язувати до реального `dt`, не до кадрів (у нас уже так — `requestAnimationFrame` з акумулятором `dt`).
- Chrome/Edge; плавно на Chromebook (4 ГБ RAM). Мишка/клавіатура/тач.
- **Мобільне:** `-webkit-user-select: none` (проти double-tap zoom); підтримка safe areas (вирізи/dynamic island); орієнтація конфігурується в сабміті; DPR=1 на iOS/слабких Android.
- **SDK:** події `Gameplay start`/`stop`; модулі `Data`/`User` за потреби; **sitelock** з whitelist доменів CrazyGames. Без SDK-інтеграції реклама заборонена.
- Privacy/terms повідомлення для збору даних (бажано non-blocking).

### Геймплей
- Гравець потрапляє в геймплей **одразу або за ≤ 1 клік**.
- Англійська — обов'язкова; інші локалі лише якісні (наш скоуп — тільки англ.).
- **PEGI 12** (контент для 13+).
- Без крос-промо (виняток — privacy/terms, посилання спільноти, сиквели серії, store-лінки на десктопі).
- Оригінальність ассетів і назв; швидке завантаження, без крашів/помилок.

### Реклама (тільки через SDK CrazyGames; зараз стаб у `src/sdk/ads.ts`, реальний SDK при релізі)
- **Midgame:** лише в логічних точках (зміна рівня/мапи, смерть), **ніколи не переривати активний геймплей** і не на кнопках навігації. Пауза + **mute звуку** на час показу; блокувати UI до `adFinished`/`adError`; порожній запит — грати далі. SDK сам обмежує частоту (≈1 на 3 хв).
- **Rewarded:** опційно, з video-іконкою і **альтернативою без реклами**; винагорода **лише на `adFinished`** (ніколи на `adError`); skip/close-кнопка збігається з reward-кнопкою (розмір/шрифт/колір); лише поза активним геймплеєм; таймери між показами, без ланцюжків заради однієї винагороди.
- **Banner:** ≥ 5 с середнього показу, не перекривати UI на жодному розмірі, ≤ 2 на екран, не під час активного геймплею.
- **Adblocker:** ніколи не блокувати гру; спец-фічі обмежувати лише з чітким повідомленням.
